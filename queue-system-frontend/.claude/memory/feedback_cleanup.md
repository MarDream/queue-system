---
name: cleanup_test_data
description: 测试后必须清理临时数据 — 用户非常在意测试残留数据污染环境
type: feedback
---

**规则：** 测试时创建的临时数据（如测试分组、测试用户、测试菜单等），在验证完成后必须主动删除或恢复到原始状态。

**Why:** 用户很反感测试数据留在数据库中，例如管理后台的 "Test Group" 被注意到后专门提醒。测试数据会让系统看起来"不干净"，影响后续开发和真实使用。

**How to apply:** 
- 每次创建测试数据验证功能后，立即通过 API 或 SQL 删除/恢复
- 在输出中明确告知用户已清理测试数据
- 如果删除操作有风险（如可能误删真实数据），先列出清理计划让用户确认
- 对于数据库种子数据，测试后确保不改动原始 seed 数据
